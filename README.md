# Azure Identity & Access Management Lab - Microsoft Entra ID

🎯 Project Overview
This lab demonstrates how to implement a secure, efficient, and scalable **Identity and Access Management (IAM)** solution in Microsoft Azure using **Microsoft Entra ID**.

The main goal is to reduce administrative overhead while maintaining strong security controls through centralized group-based management.

🛠️ What I Implemented

- Created and configured multiple users in Microsoft Entra ID  
- Organized users into a security group (**IT-Team**)  
- Assigned licenses using **Group-Based Licensing**  
- Enabled **Self-Service Password Reset (SSPR)**  
- Implemented **Role-Based Access Control (RBAC)** using group assignments  

🏗️ Architecture

Users → Security Group (IT-Team) → Role Assignment → Azure Resources**

🔐 Security & Best Practices Applied

- Centralized access management using groups (instead of individual assignments)  
- Reduced risk of human error in permission management  
- Improved user experience with Self-Service Password Reset  
- Scalable and enterprise-ready design  

✅ Key Benefits

- Simplified user and permission management  
- Reduced administrative workload  
- Enhanced security posture  
- Easy to scale when adding new users  

💡 Skills Demonstrated

- Microsoft Entra ID Administration  
- Group-Based Licensing  
- Role-Based Access Control (RBAC)  
- Self-Service Password Reset (SSPR)  
- Identity and Access Management (IAM)  
- Cloud Security Best Practices
